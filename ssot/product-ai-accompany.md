## 商品アイデンティティ

| 項目 | 内容 |
|------|------|
| 正式名称 | **AI業務再設計30日伴走プログラム** |
| 期間 | **30日間**（開始日は甲乙協議。契約第4条） |
| ゴール | 甲が**自ら AI ワークフローを構築・試行・修正**できる状態に至ること。生成AIの「使い方講義のみ」ではない（契約第3条）。 |
| スコープの芯 | **対象業務は1本（または合意した少数）**に絞る。追加・変更は**書面又は合意した通知**による合意が必要（契約第2条第2項）。 |
| 伴走の形 | **週1ミーティングでのアドバイスのみを主としない**。**現場に寄り添う「右腕」**として、ツール選定・業務範囲・実装まで当事者として関与する（契約第3条第3項・LP「市場との違い」）。 |
| 提供側の自己紹介・実績の語り | 数値は書かず、上記 `career-timeline.md` / `context.md` に合わせる。 |

## マクロ・フロー（契約第3条第2項と1対1）

本プログラムの説明・スライド・LPの「流れ」節は、次の**5ステップ**を崩さない。

1. **AI により最もインパクトが出せる業務の特定**（ボトルネック候補の洗い出し〜**対象業務の合意**まで含む）
2. **ワークフローの構築**（データ・手順・集計を含め、生成AIを前提とした形への組み替え）
3. **現場での利用**（甲の操作・検証への協力。契約第6条）
4. **エラー等の発生に応じた修正**
5. **上記一連の流れの言語化**および、甲が**自ら**実施できる状態への移行（**なぜそのツール・設計か**の説明義務。契約第3条第4項）

## 30日の推奨タイムライン（運用ガイド）

日付は**目安**。現場の密度に応じて前後してよいが、**5ステップの順序**と**伴走終了日（30日）**は契約に従う。

### 目安 Day 1〜7（ステップ1中心）

| 観点 | 内容 |
|------|------|
| 乙 | キックオフ。**業務一覧・痛み・既存ツール**のヒアリング。ボトルネック候補の整理。 |
| 甲 | 実態の開示（契約第6条）。候補業務への**同席・追加情報**。 |
| 合意事項 | **対象業務**を通知等で確定（契約第2条第2項）。 |
| 完了の目安 | 「この1本を30日でAI前提に組み替える」が双方口頭・文面で一致している。 |

### 目安 Day 8〜14（ステップ2中心）

| 観点 | 内容 |
|------|------|
| 乙 | 現状フロー・データ所在の可視化。**ツール選定**（まずは**1ツールに絞り込んで使い倒す**方針は [profile.md](profile.md) の USP と整合）。ワークフロー初版の**設計と実装支援**。 |
| 甲 | アカウント権限、サンプルデータ、**試行のための時間確保**。 |
| ツール費用 | **原則甲負担**（契約第3条第5項）。見積・契約時に扱いを明確にする。 |
| 完了の目安 | 対象業務について、**初版の手順**（人とAIの役割分けを含む）が共有されている。 |

### 目安 Day 15〜21（ステップ3〜4中心）

| 観点 | 内容 |
|------|------|
| 乙 | 実装の伴走。**エラー・想定外**に対する修正提案。 |
| 甲 | **現場で使う・壊す**。ログやエラー内容の共有。 |
| 完了の目安 | 対象業務で**少なくとも1周、意図した入力から成果が出る**体験がある（完璧は求めない）。 |

### 目安 Day 22〜28（ステップ4の継続＋ステップ5の着手）

| 観点 | 内容 |
|------|------|
| 乙 | 安定化。**判断の裏側の言語化**（選定理由・設計の意図）をドキュメント化し始める。 |
| 甲 | 自ら操作する比率を上げ、**つまずきポイント**を言語化する練習。 |
| スコープ外 | **対象業務以外の全面展開**は別合意がない限り行わない（契約第2条第2項）。 |

### 目安 Day 29〜30（ステップ5の完了）

| 観点 | 内容 |
|------|------|
| 乙 | **引き継ぎパッケージ**の完成：チェックリスト、主要プロンプト方針、ツール設定の要点、次に広げる場合の**自分用メモ**の型。 |
| 甲 | 伴走終了後も**自走で触れる**状態かの最終確認。 |
| 返金判断 | 価値実感の有無は**伴走終了時点**の甲の判断。申出期限・例外は**契約第9条**に従う（条項の数字は契約書を正とする）。 |

## 成果物の型（典型）

契約上の「成果物」の著作権は**乙帰属**・甲は事業範囲で**非独占利用**（契約第11条）。外向けに**甲の事業を特定する記述**をする場合は**第8条・別紙1**の手続き。

| 成果物の例 | 目的 |
|------------|------|
| 対象業務定義（1枚） | スコープの固定・後からの拡張判断 |
| 現状／理想の業務フロー図 | データの流れと人・AIの役割 |
| 手順書・設定メモ・プロンプト雛形 | 甲の自走用 |
| ツール選定メモ（**なぜそうしたか**） | 契約第3条第4項の言語化 |
| 既知エラーと対処一覧 | 現場での孤独を減らす |

## コミュニケーション設計（原則）

- **同期:** キックオフ、対象業務確定、中間レビュー、終了レビューを**最低限**置く（回数は案件に応じて増やしてよい）。
- **非同期:** 日々の詰まりはチャット等で**短サイクル**（乙の稼働に合わせ、返信 SLA は個別契約・案内で明示）。
- **甲の協力がないと目的達成困難**な場合、乙の責に帰さない（契約第6条第3項）— キックオフで**週あたりの投入可能時間**を擦り合わせる。

## 商い・継続（数値は `context`、条項は契約）

- **初回1ヶ月の価格・テスト枠・開始日**は `context.md`「副業：AI 導入支援」の表を**常に先に**更新する。
- **契約書上の金額・支払期日・継続時の月額（別合意がない限り金30,000円の扱い等）**は [契約書](../docs/契約書_AI業務再設計30日伴走.md) を正とする。LPの料金表記を変えたら**契約・`context`・本節の参照先の整合**を確認する。
- **経済結果の保証はしない**（契約第10条）— LP・提案でも同趣旨を崩さない。

---

## 採用しない顧客・提供拒否（商品境界）

- 一次 ICP に合わない層への**本プログラムの無理な拡張**はしない（[persona-icp.md](persona-icp.md)「ターゲット外」）。
- 提供者側の**同業（整体系など）への提供不可**は `context.md` の制約に従う（別途「対象業務」が競合に当たる場合は受注前に判断）。

---

## 保証・返金（要約のみ）

- **伴走終了時点**で価値を感じられなかった場合の**全額返金**の考え方は LP と契約で一致させる。**申出期限・営業日・例外事由**は数字まで**契約第9条**に従い、要約を変えるときは**契約書とLPを同じリリースで**直す。
